在首页只放“下载”会导致用户瞬间流失；要把**“常被问、数据可批量抓、可持续互动”**的内容端到首页，才能让访客多停留、多二次点击。下面给出两个精选模块——**一个炫酷数据榜 + 一个秒用小工具**——均满足“轻松一次性抓取 → 存本地 JSON → 前端炫展示 / 即点即算”思路。

## ① 数据展示：**流派热度榜**

### a. 数据来源

* **Fextralife Builds 页面**：收录所有主流 Build 并附带 *Votes* 字段（玩家点赞数）([Where Winds Meet][1])。
* 页面基于 MediaWiki，可直接走官方 **Parse API** 一次性抓整页 HTML，无需登录。

> **请求示例**
> `GET https://wherewindsmeet.wiki.fextralife.com/api.php?action=parse&format=json&page=Builds_(Classes_or_Path_Guides)`
> **入参**：无（固定页名）
> **返回（节选）**
>
> ```json
> {
>   "parse": {
>     "title": "Builds (Classes or Path Guides)",
>     "text": {
>       "*": "<table class=\"wiki_table\"> ... <tr><td>Nameless Sword – Swift DPS</td><td>Votes: 342</td>...</tr> ..."
>     }
>   }
> }
> ```

### b. 一次性本地化方案

1. 后端/脚本定期（手动或 CRON）请求 API，解析出 `buildName / weapon / votes`。
2. 生成 `/public/data/build_popularity.json`：

   ```json
   [
     {"buildName":"Nameless Sword – Swift DPS","weapon":"Nameless Sword","votes":342},
     {"buildName":"Panacea Fan – Bleed DoT","weapon":"Panacea Fan","votes":297}
   ]
   ```
3. 前端读取该 JSON，避免页面高频外呼。

### c. 炫酷呈现

* **横向 racing-bar 动画**（Votes 数递增滚动），或 **霓虹环形进度条**。
* 点击任一 Build → 跳到内页 `/builds/nameless-sword-swift-dps`，内页再用大模型摘要具体加点思路。

### d. 为什么能抓住用户

* “流派排名”是新手进入社区问得最频的主题([PCGamesN][2])。
* 数据每日变化小，适合离线预处理；又足够视觉冲击，天然拉停留。

## ② 轻量小功能：**秒选流派助手**

### a. 用户交互

| 步骤 | 交互形式                        | 说明                                 |
| -- | --------------------------- | ---------------------------------- |
| 1  | 下拉选择武器（12 种）                | 预填自 `build_popularity.json` 去重字段   |
| 2  | 单选玩法风格（DPS / Tank / Healer） | 减少输入成本                             |
| 3  | 点击「一键推荐」                    | 输出 3 条匹配度最高 Build（名称 + 投票数 + 核心心法） |

### b. 处理逻辑（前端 JS）

```ts
loads = fetch('/data/build_popularity.json')
filtered = loads.filter(b => b.weapon === weapon && b.role === role)
result = filtered.sort((a,b)=>b.votes-a.votes).slice(0,3)
```

### c. 输出示例

* **Nameless Sword – Swift DPS**（Votes 342）

  * 关键心法：Tiger Roar / Qi Surge
  * 一句话战术：高机动爆发，连招斩护甲

### d. 数据获取

* 依赖前述 **build_popularity.json**，无需额外外呼。
* 若需心法描述，可在首次构建时追加解析 **Best Inner Ways Guide** 页面([Where Winds Meet][3])。

### e. 现有满足方式 vs 我们的优势

* 现在玩家要么逛 Wiki 手动翻([Where Winds Meet][4])，要么看长视频测评([youtube.com][5])；**步骤繁琐、信息碎片**。
* 我们提供 **点两下即得** 且数据随排名更新，让用户形成“来本站=少走弯路”的心智。

在 **Spaceship → Cloudflare → Vercel** 这三件工具的组合里，最核心的顺序是：**先让注册商（Spaceship）把域名交给 Cloudflare 做 DNS，再让 Cloudflare 把根域／子域准确地解析到 Vercel 部署**。只要每一层职责分清、记录值无冲突，Vercel 就能立即为你的网站提供 HTTPS 与全球加速。下面给出一套“先后顺序 + 具体填写”清单，帮助你一次配好。

## 1 整体流程与角色分工

| 层级          | 工具             | 主要任务                 | 典型操作                                    |
| ----------- | -------------- | -------------------- | --------------------------------------- |
| **① 注册层**   | **Spaceship**  | 保存 whois 资料、指向外部 DNS | 把 *nameserver* 换成 Cloudflare 提供的两条 NS   |
| **② DNS 层** | **Cloudflare** | 解析记录、可选 CDN/防火墙      | 添加 Vercel 要求的 A/CNAME；通常设为 **DNS only** |
| **③ 托管层**   | **Vercel**     | 构建、托管、签发证书           | 在 *Project → Domains* 里添加并分配域名          |
